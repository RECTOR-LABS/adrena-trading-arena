Overview

TLDR

The Arena lets users mint AI trading agents as NFTs, configure strategies (RSI, EMA, Bollinger, etc.), and enter competitions where agents trade real perpetual positions on Adrena. Every trade, every P&L, every liquidation is real.


Price data comes from Pyth Hermes (real-time feeds). Indicators are computed off-chain as pure functions. The agent's on-chain account tracks ELO rating, P&L, and win/loss record — all bound 1:1 to a Metaplex Core NFT.


Right now it's SDK-level (TypeScript config). A no-code UI for non-dev traders is the natural next step. The core trading loop, on-chain program, and scoring engine are all built and tested (234 tests, 0 warnings).

User Flow

How It Works

01

Mint an Agent NFT

User creates an agent on-chain. This mints a Metaplex Core NFT and creates an Agent PDA (program-derived account) seeded from the NFT mint address. One NFT = one agent, permanently bound.

On-chain
02

Configure a Strategy

Pick indicators (RSI, EMA, Bollinger, SMA), set thresholds and risk params: max leverage, stop-loss %, take-profit %, position size %, slippage tolerance. The strategy hash is stored on-chain.

SDK today — UI next
03

Enter a Competition

Agent enrolls in a competition by paying an entry fee (SPL token transfer to a trustless prize vault PDA). Up to 256 agents per competition.

On-chain
04

Agent Trades Autonomously

The AgentExecutor runs a tick-based loop: fetch prices from Pyth → compute indicators → evaluate strategy → open/close positions on Adrena's perpetual exchange. Real trades, real P&L.

Off-chain executor
05

Score & Rank

The orchestrator monitors positions via WebSocket, calculates risk-adjusted scores (not just raw P&L), and updates the leaderboard. ELO rating adjusts across competitions.

Orchestrator
06

Claim Prize

Winner claims from the prize vault. Trustless — the on-chain program verifies final rank and releases funds. No intermediary.

Q&A

Your Questions, Answered

Q1 How are indicators calculated? From Adrena Price History or TradingView? Can they be used for backtesting?
Pyth Hermes API — real-time price feeds, no TradingView in the loop. Indicators (RSI, EMA, SMA, Bollinger Bands) are pure functions that operate on price arrays fetched from Pyth.

Currently we keep the last 1,000 price points in memory for indicator calculation. For proper backtesting, we'd need a persistent price history layer — either our own storage or hooking into Adrena's existing price history. This is a buildable next step, not a fundamental architecture change.
Q2 What if I want to combine 2 or more indicators?
Already supported. Each strategy has an evaluate(market) function that can use any combination of indicators. The Momentum strategy, for example, uses both fast and slow EMA crossover detection. You could build a strategy that checks RSI + Bollinger + EMA together — it's just logic inside the evaluate function.

The strategy interface is flexible by design — it receives full market state and returns a signal (LONG, SHORT, CLOSE, or HOLD).
Q3 Can I set agent parameters in a UI, or do I need to code?
Right now: code-level (TypeScript SDK). You define a strategy and pass a config object with risk parameters:
  • Indicator selection (RSI, EMA, Bollinger, SMA)
  • Thresholds (e.g. RSI < 30 = go long, RSI > 70 = go short)
  • maxLeverage, stopLossPct, takeProfitPct
  • maxPositionPct (% of collateral to deploy)
  • defaultSlippageBps
Your RSI agent example (5min chart, 5% collateral, 10x leverage, long <30, short >70, SL at -5%, close at RSI 50) maps directly to this config. A no-code frontend where traders fill out a form that produces this config is a straightforward next step — the architecture already supports it.
Q4 How is this different from plain algo trading? What does the NFT/Agent layer actually add?
Fair question. Without the NFT layer, it's just a bot. With it, you get:
  • Portable identity — the agent has an on-chain track record (ELO rating, total P&L, wins, losses) that lives on the NFT. It follows the asset, not the owner.
  • Composable competitions — agents enroll in competitions with entry fees and trustless prize payouts. The on-chain program handles everything.
  • Tradeable performance — if an agent has a proven track record, the NFT itself becomes valuable. A buyer inherits the strategy + verified performance history.
  • Native to Adrena — agents open/close positions through Adrena's actual instructions. They're first-class participants in the ecosystem, not external bots.
Q5 What's the link between my agent, the Metaplex NFT, and my wallet?
  • NFT mint → A Metaplex Core NFT is created during agent creation (new keypair).
  • Agent PDA → On-chain account seeded from the NFT mint address: [AGENT_SEED, mint]. This is a 1:1 binding — can't be separated.
  • Agent account stores → owner pubkey, mint address, strategy_hash, elo_rating, total_pnl, wins, losses, status.
  • Wallet → Currently the owner's wallet signs all trades directly. A dedicated agent PDA wallet (for capital isolation — "allocate X SOL to this agent") is a natural next step.
Transfer the NFT = transfer ownership of the agent and its entire track record.
Q6 If the strategy is embedded in the NFT, can the owner change it before selling? Freeze authority?
The strategy_hash is stored on-chain in the Agent account. This means anyone can verify what strategy an agent is running by comparing the hash.

For a marketplace to work with trust, we'd need:
  • Freeze authority on the strategy — once frozen, the strategy can't be modified. This is architecturally supported but not yet implemented.
  • On-chain verification that the running strategy actually matches the stored hash.
  • A clear flow: create agent → prove performance → freeze strategy → list for sale.
The primitives are there (hash storage, ownership transfer via NFT). The freeze + verification layer is buildable.
Q7 Can I allocate capital to an agent separately? On/off switch? Sub-wallet?
Not yet, but the architecture supports it. The current model has the owner signing trades directly with their own wallet. The vision you're describing — "I allocate 5 SOL to this agent, it trades with that, I can turn it on/off" — would need:
  • A dedicated agent vault (PDA-controlled token account) where users deposit capital
  • Delegation so the executor can trade on behalf of the agent without the owner signing every tx
  • An on/off status on the agent account (the status field already exists)
This is the most impactful next feature for making it feel like a real product rather than a dev tool.

Architecture

Anchor/Rust. Manages agents, competitions, enrollments, prize vaults, and scoring. Single source of truth for all state.

TypeScript SDK

Client library for interacting with the program. Contains strategies, indicators, AgentExecutor, LiveAdrenaTrader, and price feed integration.

TypeScript Pyth Hermes Vitest
Layer 3

Orchestrator

Rust backend service. Manages competition lifecycle, monitors positions via WebSocket, calculates scores, and maintains the leaderboard.

Rust WebSocket PostgreSQL

Frontend

Next.js web app. Dashboard for creating agents, entering competitions, viewing leaderboards, and monitoring live performance.
